<threat_model>
This plan creates core data structures. Main risks are data corruption, unstable exports, and unsafe mutable references.

- Threat: invalid way references corrupt exported data. Mitigation: primitive store validates node references and closed-way sequences before export.
- Threat: tests become flaky due to real clock/time. Mitigation: all new timestamps come from an injectable clock.
- Threat: local IDs collide across element types. Mitigation: allocator uses separate large numeric ranges for node, way, and relation IDs.
</threat_model>
