<threat_model>
This plan connects MapLibre events to core mutation flows. Main threats are wrong hit-test routing, drag state leaks, topology regression, and false parity from adapter-only tests.

- Threat: layer clicks select or insert the wrong feature. Mitigation: use GeoJSON properties `featureId`, `vertexIndex`, and `edgeIndex`; tests fire layer events with those properties.
- Threat: map drag-pan remains disabled after SDK drags. Mitigation: restore on pointer up, cancel, and detach; tests inspect fake `dragPan` calls.
- Threat: MapLibre edits bypass shared topology semantics. Mitigation: editor integration tests assert primitive node/way updates and shared-node behavior through exports.
- Threat: snapping visuals pass but snapping data fails. Mitigation: draw adjacent rooms through MapLibre with snapping enabled and assert exported shared node IDs.
- Threat: host-owned UI boundary erodes. Mitigation: tests use host-style editor API calls for tool/finish/cancel/tag flows, not SDK UI controls.
</threat_model>
