<threat_model>
This plan touches public API and host integration semantics. Main threats are breaking existing editor options, emitting renderer-specific payloads, swallowing errors, and turning the SDK into a UI framework.

- Threat: existing callers using `ids`, `snapping`, or `validationRules` break. Mitigation: add new options compatibly and preserve existing option names.
- Threat: lifecycle/error events leak MapLibre event shapes into core. Mitigation: keep `EditorEventMap` payloads renderer-neutral; adapter original events remain adapter-level opaque values only.
- Threat: `ready` or `error` events are impossible to observe because they emit too early or not at all. Mitigation: emit `ready` asynchronously after construction and wrap adapter-driven editor mutations to emit `error` before rethrowing.
- Threat: default tags override required indoor/level tags. Mitigation: apply default tags before per-draw host tags and keep `createMinimumTags` enforcing required `indoor` and `level` values.
- Threat: style overrides require hosts to learn core internals. Mitigation: keep MapLibre style API in `packages/maplibre` with role-based keys and partial merge tests.
</threat_model>
