<threat_model>
This plan evaluates user/imported geometry and tags. Main threats are unsafe assumptions about malformed data, expensive validation, and hosts misinterpreting advisory output as an SDK save gate.

- Threat: validation crashes on malformed but structurally loaded elements. Mitigation: each rule returns issues and does not throw except for programmer errors.
- Threat: large geometry validation becomes slow. Mitigation: rules operate on current in-memory primitives and modular Turf checks only where needed.
- Threat: export becomes blocked by validation. Mitigation: tests assert `exportOsmInEdit()` still returns data when validation issues exist.
- Threat: custom rules mutate editor state. Mitigation: validation context exposes readonly snapshots/copies.
</threat_model>
