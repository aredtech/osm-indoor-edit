<threat_model>
This plan turns internal project state into public-facing docs and release scripts. Main threats are publishing accidentally, documenting features outside v1, breaking package boundaries, and creating a release gate that omits examples.

- Threat: release automation publishes packages accidentally. Mitigation: `release:dry-run` must not run `npm publish` or `pnpm publish`; docs state actual publishing is manual.
- Threat: docs imply backend, real OSM upload, framework wrappers, or floor-plan calibration are v1. Mitigation: public boundary docs list these as out of scope.
- Threat: core package pulls renderer dependencies into all consumers. Mitigation: package metadata and `check-core-boundary` verify no renderer imports/deps in core.
- Threat: release gate ignores examples. Mitigation: dry-run includes `build:examples`, `test:e2e:leaflet`, and `test:e2e:maplibre`.
- Threat: package metadata omits public entrypoints. Mitigation: add explicit `exports`, `types`, `files`, and side-effect metadata for core and adapter packages.
</threat_model>
