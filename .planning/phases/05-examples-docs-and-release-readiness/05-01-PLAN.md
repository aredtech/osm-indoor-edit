<threat_model>
This plan creates public examples that developers will copy. Main threats are implying SDK-owned application UI, depending on external map tiles, making validation look blocking, or creating renderer behavior drift.

- Threat: examples imply the SDK ships a fixed toolbar/sidebar. Mitigation: keep all buttons/panels inside `examples/*` and call editor APIs directly from host code.
- Threat: examples or Playwright smoke depend on external tile/network services. Mitigation: keep Leaflet tile-free and initialize MapLibre with an inline local style.
- Threat: validation appears to block export. Mitigation: render validation issues separately and leave export controls usable.
- Threat: Leaflet and MapLibre examples drift. Mitigation: use shared fixture/helpers and matching Playwright assertions for both examples.
- Threat: shared utilities become a hidden UI framework. Mitigation: limit `examples/vanilla` to data/helpers, not controls or layout harness.
</threat_model>
