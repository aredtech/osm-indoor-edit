<threat_model>
This plan exposes preset workflows in public examples and docs. Main threats are implying SDK-owned UI, breaking mobile panel layout, documenting APIs that do not compile, and losing renderer parity.

- Threat: example picker/form looks like an SDK component. Mitigation: keep all markup and CSS in `examples/*`, and docs state hosts own UI.
- Threat: examples become cramped or overlapping on mobile. Mitigation: follow `06-UI-SPEC.md` panel width, spacing, and scroll constraints.
- Threat: docs drift from actual API names. Mitigation: compile examples and run `pnpm typecheck`.
- Threat: Leaflet and MapLibre examples diverge. Mitigation: implement matching visible labels and matching Playwright assertions.
- Threat: examples hard-code field tags instead of using SDK schema/helpers. Mitigation: example form labels and field application must read preset fields from the SDK catalog.
</threat_model>
