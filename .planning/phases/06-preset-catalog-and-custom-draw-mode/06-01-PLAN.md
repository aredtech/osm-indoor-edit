<threat_model>
This plan introduces public preset data and helper APIs. Main threats are runtime XML parsing, catalog scope bloat, mutable built-in data, unsafe tag deletion, preset metadata leaking into OSM tags, and package asset mistakes.

- Threat: runtime XML parsing bloats browser bundles and violates D-02. Mitigation: ship normalized TypeScript data only; acceptance checks reject `xml2js` and XML parser references.
- Threat: full `default.xml` parity expands Phase 6 too far. Mitigation: use a curated common subset with motorcycle dealer and representative amenity/shop/office entries.
- Threat: host overrides mutate built-in data globally. Mitigation: clone preset data when creating catalog instances and test that built-ins remain stable.
- Threat: tag helpers remove hard tags accidentally. Mitigation: `TagDiff` exposes `protectedKeys`, and sparse field helpers preserve hard preset tags.
- Threat: icons force UI ownership. Mitigation: expose SVG strings/asset IDs only; no picker components or rendering code.
</threat_model>
