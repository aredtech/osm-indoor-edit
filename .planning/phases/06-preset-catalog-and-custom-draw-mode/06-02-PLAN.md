<threat_model>
This plan changes the editor draw lifecycle. Main threats are breaking existing draw kinds, creating invalid open/closed ways, applying incompatible presets, mutating primitives before validation fails, and leaking preset metadata into exported OSM elements.

- Threat: existing `room`, `corridor`, and `poi` behavior regresses. Mitigation: keep existing tests and add custom tests without changing their expected exports.
- Threat: custom line drawing accidentally creates closed ways. Mitigation: explicit `geometryType: "line"` finish path calls `createWay({ closed: false })`.
- Threat: invalid preset geometry starts a draft and leaves temporary state. Mitigation: validate preset ID and geometry before assigning `this.draft`.
- Threat: preset metadata leaks into OsmInEdit JSON tags. Mitigation: store metadata on `FeatureRecord.preset`, not in primitive tags; import/export tests assert no `presetId` tag.
- Threat: sparse tag updates break existing `updateTags` merge-only API. Mitigation: add a separate diff application path and leave `updateTags` behavior intact.
</threat_model>
